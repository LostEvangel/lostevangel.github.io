# 我们做了一个公众号排版 Skill

## 起因

我习惯用 Markdown 写稿，写到公众号上就需要转成好看的 HTML。市面上有不少排版工具，但每次都要手动复制粘贴、调整样式，流程割裂。

正好 Claude Code 有 Skill 机制——可以自定义一套自动化工作流，用自然语言触发。于是想：能不能做一个排版 Skill，一句话就把文章排好？

成品就是 `/wechat-article-beautify`。

---

## Skill 设计：四套主题

排版的核心是风格。我们设计了一套主题样式库，每套都有完整的配色和排版规范：

| 主题 | 配色基调 | 适合 |
|------|---------|------|
| 🍃 **极简清风** | 白底灰字，深蓝标题 | 通用文章，最保险 |
| 📖 **书香典雅** | 米白底，棕褐标题 | 人文随笔、书评 |
| 🌸 **温暖杂志** | 暖橙粉紫，圆角卡片 | 生活分享、情感 |
| 💻 **技术格调** | 深色代码块，硬朗 | 技术教程 |

以我最常用的 **书香典雅** 为例，它的核心设定是：

```
背景: 米白 (#FDFBF7)        标题: 棕褐 (#8B4513)
正文: 深灰 (#3D3D3D)        引用: 暖色左边框 + 浅暖底
间距: 行高 1.75，段距 20px
装饰: ❦ 花式分隔符，首字下沉
```

每套主题都包含完整的样式规则：标题层级、引用块、代码块、列表、分隔线、图片，覆盖一篇公众号文章可能用到的所有元素。

---

## 工作流：五步走

Skill 的工作流程设计为五个阶段，用户只需要提供内容和选择主题，剩下的自动完成：

**Step 1 — 选主题**：列出四套风格让用户挑，或直接指定

**Step 2 — 分析结构**：识别 Markdown 中的标题层级、引用、列表、代码块、加粗强调

**Step 3 — 生成 HTML**：按主题规范输出全部内联样式的 HTML 文件。这步最核心——要处理标题的装饰线、引用块的左边框和背景色、分隔符的样式、列表的缩进等等

**Step 4 — 输出结果**：告诉用户字数、阅读时间，附 HTML 文件路径和使用说明

**Step 5 — 可选润色**：如果用户需要，帮忙优化内容表达

流程不复杂，但 Step 3 踩了大坑。

---

## 最大的坑：微信编辑器不认 CSS

HTML 生成后在浏览器里打开，效果很好。但一旦全选复制、粘贴到微信公众号编辑器，样式就大面积阵亡。

一轮轮测试下来，我们摸清了微信的"脾气"：

### 微信不认的

- **整体背景色**：微信强制白色，无解
- **CSS `border` 系列**：`border-left`、`border-top` 等全部过滤
- **`bgcolor` 属性**：HTML 背景色属性一样过滤
- **`letter-spacing`**：字间距不保留
- **块级元素字号**：`<div>` 上的 `font-size` 不生效
- **`font-weight`**：必须用 `<b>` 标签
- **`float` / `position` / `border-radius`**：布局和圆角属性全删

### 微信认的写法

| 效果 | 正确写法 |
|------|----------|
| 文字颜色 | `<font color="#xxx">`（不是 CSS `color`） |
| 加粗 | `<b>` 或 `<strong>` |
| 居中 | `<center>` 标签 |
| 横线 | `<hr style="border: none; border-top: 1px solid #ccc;">` |
| 标题 | `<h1>` ~ `<h6>` 语义标签 |
| 行高 | `<p>` 上写 `line-height` |
| 缩进 | `text-indent: 2em` |

核心经验就一句话：**能用 HTML 标签解决的问题，就不要用 CSS。**

---

## 踩坑实例：左边框装饰条

小标题前的棕色竖条是我们最纠结的一个细节。

最初的方案是用 `border-left: 4px solid`，浏览器里很优雅。微信过滤后光秃秃的。

尝试用 Unicode 字符 `▎`（LEFT ONE QUARTER BLOCK）加上颜色——微信不渲染这个字符，直接消失。

换成 `‖`（双竖线）——渲染了，但视觉上不如实心色块。

最终结论：这种纯 CSS 装饰在微信里确实留不住，要么接受降级，要么用字符近似替代。我们选择了前者——在 Skill 的主题设计时就主动放弃依赖 `border` 的装饰方案。

---

## 沉淀到 Skill 里

经过这轮实战，我们对 Skill 做了几项关键更新：

1. **内置兼容性清单**：生成 HTML 时自动规避被微信过滤的写法，优先使用标签方案
2. **字号策略调整**：正文 15px 替代 16px，小标题 17px 替代 19px，更贴近微信实际渲染效果
3. **降级设计**：主题设计时主动放弃注定丢失的样式（整体背景色、边框装饰、圆角），保住阅读体验核心
4. **预期管理**：输出时主动告知用户"浏览器预览 ≠ 微信效果"

---

## 未来规划

Skill 目前能完成基础的排版工作，但还有两个方向可以做得更深：

### 1. 排版再优化

当前排版依赖预定义的主题模板，后续可以探索更灵活的方式：

- **精细化控制**：允许用户微调单个参数（字号、间距、配色），而不是只能选整套主题
- **更多主题**：增加新的风格，比如适合诗歌的留白极简风、适合影评的胶片色调
- **WeChat 降级策略自动化**：在生成 HTML 时自动检测并替换被微信过滤的写法，减少手工调整
- **移动端预览**：生成时附带移动端预览截图，减少"浏览器 ≠ 微信"的预期落差

### 2. 文案优化

排版之外，写作本身也有优化的空间：

- **风格化润色**：用户写完草稿后，Skill 可以根据文章类型（技术/随笔/书评）自动调整语气和措辞
- **长段落切分**：识别过长的段落，拆分为更适合手机阅读的短段
- **节奏感优化**：分析句长分布，建议在需要停顿的地方插入分隔或小节标题
- **多风格适配**：同一篇内容可以按"正式版"和"口语版"分别输出，适配不同发布渠道

这两个方向一个偏"形式"，一个偏"内容"，我们希望 Skill 最终能兼顾两者——不仅让文章好看，也让文章好读。

---

## 收获

这次做 Skill 最大的收获不是那张兼容性清单，而是一个认知：

**公众号排版是一个降级过程，不是升级过程。**

Markdown → 精美 HTML 是"设计态"，HTML → 微信编辑器是"运行态"。从设计态到运行态，注定要打折扣。Skill 要做的不是对抗这种降级，而是在设计时就预判哪些能保、哪些该放，用最小的代价保住阅读体验的核心。

如果你也在折腾公众号排版，希望这份记录对你有帮助。
